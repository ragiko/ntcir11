次の図は、共有アドレス スペースを構成する前の University of Fabrikam のメール ルーティングを表しています。
ここで注目すべき点は、共有アドレス スペース @fabrikam.edu が既に社内メッセージング システムによって権限のあるドメインとして使用されていることです。
電子メールを最初に Outlook Live に配信する共有アドレス スペースを University of Fabrikam で構成するには、次の手順を実行する必要があります。
社内メッセージング システムに campus.fabrikam.edu を追加し、@campus.fabrikam.edu プロキシ アドレスを使用してメールボックスを構成する
クラウドベースの電子メール サービスに fabrikam.edu を登録する
クラウドベースの電子メール サービスに campus.fabrikam.edu を承認済みドメインとして追加する
クラウドベースの電子メール サービスで campus.fabrikam.edu を内部の中継ドメインとして構成する
クラウドベースの電子メール サービスで、社内メッセージング システムのユーザーを表すメール ユーザーを構成する
社内メッセージング システムで、fabrikam.edu を権限のないドメインとして構成し、@fabrikam.edu の受信者のクラウドベースの電子メール サービスへのルーティングを構成する
fabrikam.edu の MX レコードを、クラウドベースの電子メール サービスを指すように構成する
クラウドベースの電子メール サービスでユーザーのメールボックスを作成する
最終的に共有アドレス スペースをクラウドベースの電子メール サービスで権限のあるドメインとして構成できるようにするために、社内ドメインを追加する必要があります。ドメインが複数のメッセージング システムで権限のあるドメインになっていると、メール ルーティング ループが発生します。University of Fabrikam の例では、fabrikam.edu が社内メッセージング システムで権限のあるドメインになっているため、新たに campus.fabrikam.edu を社内メッセージング システムに追加して、権限のあるドメインとして構成する必要があります。その際、社内メッセージング システムで構成されている fabrikam.com ドメインは、そのままにしておきます。これにより、@fabrikam.edu の電子メールの配信をクラウドベースの電子メール サービスに移動するまでの間、@fabrikam.edu の受信者に送信された電子メールを社内メッセージング システムに配信できます。
このステップの具体的な手順は、使用している社内メッセージング システムによって大きく異なります。次にいくつかの例を示します。
Microsoft Exchange Server 2010 および Exchange Server 2007 の場合次の手順を実行します。
Exchange 管理コンソールまたは Exchange 管理シェルを使用して、新たに campus.fabrikam.edu を、権限のある承認済みドメインとして追加します。詳細については、「承認済みドメインの作成」 (Exchange 2010) または「承認済みドメインを作成する方法」 (Exchange 2007) を参照してください。
新しい電子メール アドレス ポリシーを作成および適用して、すべてのメールボックスに @campus.fabrikam.edu プロキシ アドレスを追加します。既定の返信アドレスや承認済みドメインは @fabrikam.edu のままにしておきます。詳細については、「電子メール アドレス ポリシーの管理」 (Exchange 2010) または「電子メール アドレス ポリシーの管理」 (Exchange 2007) を参照してください。
Exchange Server 2003 の場合Exchange が権限を持つ名前空間を共有することはできません。University of Fabrikam の例では、既定の受信者ポリシーのプライマリ SMTP アドレスを @fabrikam.edu から @campus.fabrikam.edu に変更する必要があります。手順については、マイクロソフト サポート技術情報の記事番号 321721「Exchange 2000 Server または Exchange Server 2003 で SMTP アドレス スペースを共有する方法」 (方法 1 の手順 1 および 2) を参照してください。
社内メッセージング システムで campus.fabrikam.edu ドメインを構成できたら、インターネットに接続された DNS サーバーで campus.fabrikam.com の MX レコードを構成する必要があります。共有アドレス スペースと社内ドメインの両方のメール フローが社内ユーザーに対して機能することを確認してください。電子メール アドレス @campus.fabrikam.edu は、インターネット上の送信者は使用しませんが、クラウドベースの電子メール サービスが電子メールを社内メッセージング システムにルーティングするために使用します。
この段階では、社内メッセージング システムで共有アドレス スペース fabrikam.edu を権限のあるドメインのままにしておきます。最終的には、メール ルーティング ループを防ぐために、社内メッセージング システムで fabrikam.edu のドメインの種類を権限のあるドメインから権限のないドメインに変更します。
この手順が完了すると、University of Fabrikam のメール ルーティングは次のようになります。
