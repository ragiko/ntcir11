Windows Vista 早期採用者1 から得たレッスン
オペレーティング システムの移行は、ほとんどだれもができれば避けたいと思うことの 1 つです。新しいアプリケーションを導入する場合とは異なり、OS の移行は PC エコシステムのすべての側面に影響し、負担が大きく時間がかかる作業です。PC 環境全体に影響が及ぶため、すべての要素を個々にテストし、入念に計画を立て、周知を徹底することが重要です。どんなに経験を積んだ IT プロフェッショナルでも、他のだれかが先にやり遂げ、注意が必要な点を教えてくれればと願うのは当然のことです。
ただし、オペレーティング システムは PC エコシステムのバックボーンであり、ときどき環境が更新されることは避けられません。Windows Vista は、インフラストラクチャ管理がコア機能として組み込まれた初の PC オペレーティング システムです。導入プロセスは簡単ではありませんが、長期的には、その負担を埋め合わせて余りある価値をもたらします。
このドキュメントでは、Windows Vista を組織に展開するうえで特に難作業となる問題への対処方法について説明します。Windows Vista には Windows XP とは大きく異なる点がいくつかあるため、慣れるまでに多少苦労する可能性があることは確かです。ここでは、既に Windows Vista を導入している「最前線」の組織から集めた教訓をもとに、実際に発生した問題とその有効な解決策を紹介します。
課題 1: ハードウェアの互換性
率直に言うと、Windows Vista では正常動作のために従来のバージョンの場合よりはるかに堅牢なハードウェアが必要です。特に、Windows Aero™ インターフェイスの高度な視覚効果を生かすためには、ハードウェアの性能が重要になります。規定の最低要件はごく標準的 (512MB RAM) ですが、実際には、ほとんどの場合、Windows Vista を実行する PC には約 2GB の RAM、高性能ビデオ カード、デュアルコア プロセッサが必要となります。
対処方法
まず、Windows Vista Upgrade Advisor または (複数のシステムを評価する場合は) Windows Vista Hardware Assessment (WVHA) Tool をダウンロードして実行します。
Upgrade Advisor は 1 つのスタンドアロン システムの互換性を報告するツールであり、ハードウェアおよびソフトウェアの互換性に関する詳細なレポートが生成されます

WVHA は、インベントリ、評価、レポート作成の機能を備えたツールで、Windows Vista を早期に導入した IT プロフェッショナルからのフィードバックをもとに設計されています。このツールでは、ネットワーク上のコンピュータを検出し、Windows Vista を実行するための条件が整っているかどうかを判定できます。Windows Vista ハードウェア評価ウィザードは、評価結果の概要と詳細を示すカスタム レポートを生成し、PC が Windows Vista に対応できるかどうかのベースラインと、変更または別の展開方法を必要とする PC を示します。
ほとんどの場合、Windows Vista を導入するときにはハードウェアを更新する方が安全です。Windows Vista から最も大きなメリットが得られるユーザー グループ (モバイル ユーザーやナレッジ ワーカーなど) を特定したら、それらのグループを対象として予算内で新たなハードウェアを購入し、各グループを段階的に Windows Vista へ移行します。次に、ハードウェア予算が確保でき、古い PC が耐用年数に達したら、Windows Vista の必要性がそれほど高くない残りのグループを移行します。Windows Vista は Windows XP と共存でき、Windows Server ドメインを結合できるほか、グループ ポリシーを使用した構成や管理も可能です。システムのアップグレードが進み、最終的に Windows XP を実行する PC がなくなった時点で、従来の Windows XP グループ ポリシー設定を Active Directory から削除できます。
Windows Vista のセキュリティおよび管理の側面が環境と適合し、Windows XP を上回るレベルの制御が可能となる状況があります。たとえば、コール センターと小売店舗をロックダウン環境で管理する場合、IT 管理者は一般に、サービスの停止につながるイベントを防止し、エンド ユーザーの生産性を維持するために必要なレベルの権限を持ちます。Windows Vista ではグループ ポリシーの機能が拡張されており、電力管理を含め、設定および構成に対して Windows XP より詳細な制御が可能です。このような場合、ハードウェア (通常は RAM) のアップグレードに要するコストは、生産性とセキュリティの向上によって相殺できます。
Windows Vista のハードウェア要件を満たす有効な方法の 1 つとして、PC を端末またはシン クライアントと位置付け、処理を中央サーバーでホストすることがあります。これによって、ユーザーは、ソフトウェアをローカルに実行できないハードウェアから Windows Vista の仮想デスクトップや x86 ベースの Windows アプリケーションにアクセスできます。この方法は、通常は高性能の PC が使用できるようになるまでの短期的な解決策ですが、エンド ユーザーが既存のハードウェアとアプリケーションをそのまま使用できるというメリットがあります。
課題 2: ビジネス アプリケーション
アプリケーションの互換性は、新しいオペレーティング システムへ移行する際に発生する難問の 1 つです。Windows XP 向けのアプリケーションのほとんどは Windows Vista でも正常に動作しますが、ビジネスに不可欠なアプリケーションには、更新しなければ正しく動作しないものが多く存在します。この問題は、Windows Vista を導入する際の障害となりそうに思えますが、早期採用者が使用した手法の中に、この問題を解決するヒントとなるものがあります。
対処方法
まず、アプリケーション ポートフォリオを示す詳細なインベントリを作成し、各アプリケーションの状態を調べます。Microsoft では、この作業を支援するツールとして、Application Compatibility Toolkit (ACT) を開発しました。このツールでは、以下のことを実行できます。
アプリケーション、Web サイト、およびコンピュータを分析する。
Windows Vista がアプリケーションおよび Web サイトに与える影響を評価する。
アプリケーション互換性の問題に対する取り組みの優先順位を決定する。
互換性に関する既知の問題に対する自動化された軽減策を展開する。
Application Compatibility Toolkit では、Microsoft、ISV、および ACT コミュニティから情報を収集する Web サービスである Application Compatibility Exchange との間で、互換性に関する最新情報を送受信することもできます。さらに、Microsoft では、企業のお客様に低コストのアプリケーション互換性検証および修正サービスを提供するパートナーを紹介する Application Compatibility Factory (ACF) も作成しました。
次に、Windows Vista を展開する時点で互換性がない重要なアプリケーションに対処します。この問題の解決に成功した組織が使用した方法は、互換性のあるオペレーティング システム (Windows 2000 または Windows XP) の仮想インスタンスを作成し、Microsoft Virtual PC 2007 を使用して Windows Vista PC でホストするというものです。
課題 3: 社内で開発されたアプリケーション
社内の開発チーム が作成した基幹業務アプリケーションの場合、リード タイムやリソースが要因となり、Windows Vista を展開したときに互換性のあるバージョンを提供できなくなることがあります。この問題は前の課題と似ていますが、解決策が社内のリソースに依存するため、異なる方法を使用する必要があります。
対処方法
最も重要な対策は、社内の開発チームを早い段階から Windows Vista の展開計画に参加させることです。開発チームは、プロジェクト計画にバージョンの更新を盛り込み、開発スケジュールが決まりしだい直ちに取り掛かる必要があります。
この問題でも、まず Application Compatibility Toolkit を使用します。ACT 5.0 では、社内で開発されたアプリケーションおよび Web サイトのうち、動作しないもの、軽微な問題があるもの、問題がないものを特定でき、さらに開発者向けの新しいツールを使用して、セットアップ パッケージ、Web サイト、および Web アプリケーションを Windows® Internet Explorer 7 でテストして、Windows Vista で標準ユーザーとして実行した場合に発生する問題を検出します。
課題 4: ユーザー アカウント制御
Windows Vista の新しいセキュリティ機能であるユーザー アカウント制御 (UAC) に問題が伴うことはよく知られています。この機能は、悪意のあるソフトウェアがユーザーの PC に侵入することを防ぐ働きをします。Apple Computer が "PC Guy" を使った広告でこの機能を揶揄したこともありました。しかし、UAC は、IT プロフェッショナルの PC 環境に対する制御を強化することにつながる新しい強力なセキュリティ機能です。
Windows Vista では、すべてのユーザーに標準ユーザー モードでの実行を強制することにより、ユーザーが明示的な承認を得ることなくコンピュータの不安定化やネットワークへのウイルスの侵入を招きかねない変更を行う能力を最小化します。その結果、ヘルプ デスクへの緊急の問い合わせが減少し、マルウェアの影響を軽減でき、さらに共有コンピュータ上の機密データを保護できます。
IT プロフェッショナルが解決しなければならない問題は、管理者特権を必要とするプロセスを含む従来のアプリケーションが数多くあり、それらのプログラムがファイル、レジストリ キー、またはオペレーティング システムの設定にアクセスし、変更を加える可能性があることです。中断を回避し、アプリケーションの安定した動作を保証するためには、それらのアプリケーションが標準ユーザー モードで動作するように設定する必要があります。
対処方法
UAC に伴う問題の回避には 2 つの側面があります。1 つは、エンド ユーザーに対して UAC プロンプトに応答する方法を教えることです。もう 1 つは、アプリケーションを標準ユーザー モードで動作するように設定することです。Windows Vista では、ファイル システムとレジストリ空間の仮想化によって、従来のアプリケーションを標準ユーザー アカウントで実行できます。従来のアプリケーションを設定するときには、管理者権限で動作することを前提とするグローバル アプリケーション インストーラによって、アプリケーションの実行可能ファイルおよび補助データを保存するディレクトリとアプリケーション設定のキーを作成することをお勧めします。このプロセスの詳細については、『TechNet Magazine: Windows Vista ユーザー アカウント制御の内部』を参照してください。
課題 5: エンド ユーザーの準備
エンド ユーザーが作業を行う PC 環境を変更すると、ヘルプ デスクに問い合わせが集中し、技術に詳しくないユーザーの間で混乱が発生することは避けられません。慣れ親しんだタスクの実行方法が急に変わるからです。ほとんどの場合、変更は簡単に対応できるようなものですが、ユーザーによっては習得に時間がかかることもあります。もちろん、使いやすさや一般的なタスクの合理化について徹底的な研究を行ったうえで変更を加えていますが、何かが少し移動しただけで混乱を引き起こす場合もあります。
対処方法
混乱を防止し、ヘルプ デスクへの問い合わせを少なくし、速やかに生産性を向上させる最善の方法は、新しい環境での一般的なタスクの実行方法を早い段階から頻繁に周知しておくことです。プロジェクト計画にエンド ユーザー トレーニングを盛り込み、学習内容を理解しやすい小単位に分けて少しずつ教えていく必要があります。
大規模なグループの移行に成功した組織では、エンド ユーザーが新しい PC 環境を使用し始める 1 日目から何が変わるかを把握できているように、展開プロセスの早い段階で基礎を作り上げています。変更の概要を示すだけでなく、一般的なタスクを実行するうえでの「変更トップ 10」のようなリストも配布しています。それと同様に重要であるのは、新しい環境がなぜ生産性を高めることになるのかを知らせることです。エンド ユーザーの支持を確かなものにすることで、変化を受け入れ、個人が状況を把握しようとする義務感を持つようにする必要があります。このプロセスを進めるうえで役に立つ資料として、「エンタープライズ ラーニング フレームワーク」(英語) があります。エンタープライズ ラーニング フレームワーク (ELF) は、Windows Vista および 2007 Microsoft Office system の展開プロセス中に実施する従業員トレーニングおよびコミュニケーション計画を作成するための企業向けツールです。ELF では、展開の段階ごと、およびユーザーの種類ごとに、Windows オンライン ヘルプと Office Online から最も関連性の高いトピックを選び出すことができます。ELF には次のようなメリットがあります。
展開へ向けて従業員の準備を整え、新バージョンのメリットを周知することにより、不安を軽減できる。
簡単な留意事項リストを配布して従業員に周知徹底することにより、展開の実行日の混乱を最小限に抑えることができる。
従業員が Windows Vista と 2007 Microsoft Office system を最大限に活用するうえで役に立つヒントと秘訣や生産性の向上に関するトピックを選択できる。
展開に成功した組織は、移行の実行日に、新機能を挙げると共に一般的なタスクと新機能の両方の操作手順を説明した案内メールを送信しています。資料が揃っている場合は、簡単なハウツー ガイドを作成すると、新しい環境を使い始めるユーザーに安心を与えるうえで大いに役立ちます。
