この資料では、ネットワーク プロトコルとして TCP/IP を使用しているときに発生する可能性のある、一般的なネットワーク通信問題の対処方法を説明します。問題は通常、以下の 2 つのカテゴリのうちの 1 つになります。
特定の IP アドレスに接続できない
特定のホストまたは NetBIOS 名に接続できない
特定の IP アドレスに接続できない場合、問題は基本的な接続性に関係します。特定の IP アドレスに接続できるが、その IP アドレス用のホストまたは NetBIOS 名を使って接続できない場合には、問題は名前解決方法に関係します。
問題が基本的な接続性に関係しているのか、あるいは名前解決方法に関係しているのかを判定するために、以下の手順を使って、特定の IP アドレスに接続できるかどうかを判断します。
IP アドレスへの接続
IP アドレスと TCP/IP プログラムまたは選択したユーティリティを使って、ネットワーク上の ほかのコンピュータへ接続してみます。Web ブラウザ、ftp、および Telnet は、TCP/IP で別のコンピュータへ接続するために一般的に使用されているユーティリティです。
注 : 接続したい Windows NT コンピュータの IP アドレスがわからない場合には接続したいコンピュータ上で、コマンド プロンプトから IPCONFIG / ALL コマンドを実行します。
IP アドレスを使ってほかのコンピュータに接続できない場合、基本的な接続性に問題があります。以下の「特定の IP アドレスへ接続できない」に書かれている情報を利用して、問題を解決します。IP アドレスを使ってほかのコンピュータに接続できるけれども、ほかのコンピュータのホスト または NetBIOS 名を使って接続できない場合には、おそらく名前解決方法に問題があります。以下の「特定のホストまたは NetBIOS 名へ接続できない」に書かれている情報を利用して、問題を解決します。

TCP/IP 構成のチェック
ネットワーク プロトコルとして TCP/IP を使っている場合、無効な TCP/IP 設定 (無効な IP アドレスまたは無効なサブネット マスクなど) によって、通信障害が発生する可能性があります。Windows NT または 2000 のどちらかで、不適切な TCP/IP 設定によるエラーを記録するかを決定するには、イベント ビューアで表示されるシステム ログを調べ、TCP/IP または DHCP をソースとするエントリを検索します。イベント ビューアのエントリを表示するためには、そのエントリをダブルクリックします。
注 : イベント ビューアに DHCP のエラーが記録されている場合、そのエラーをネットワーク管理者へ報告する必要があります。
イベント ビューアのシステム ログで TCP/IP エラーが記録されている場合には、記録されている各エラーを、エラー メッセージに示されているように解決します。例えば、IP アドレス パラメータが正しくないというエラーが記録されている場合、ユーザーの IP アドレスが有効であることを確認する必要があります。
イベント ビューアのシステム ログ内にエラーがない場合、以下の手順に従い、正しい TCP/IP 構成情報が使用されているかを確認します。
IPCONFIG コマンドを使い、ユーザーのコンピュータの基本 TCP/IP 設定を確認します。これを行うには、コマンド プロンプトで "ipconfig" (引用符を除く) とタイプします。
IPCONFIG コマンドによって表示された IP アドレスとサブネット マスクが、ユーザーのコンピュータに対して正しい値であるか確認します。正しい値が明確でない場合には、ネットワーク管理者に問い合わせます。
