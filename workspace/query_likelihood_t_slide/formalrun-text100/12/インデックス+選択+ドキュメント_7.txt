
ドキュメントの参照    概要
Curl IDE のドキュメントには、Curl言語を効率良く学習できるよう、リンクや インタラクティブな例題などの機能が含まれています。Curl IDE のドキュメント ビューアは、これらのドキュメント表示を目的として Curl で作成された、ブラウザに似た特別なアプリケーションです。
ドキュメントにはリンクが含まれています。たとえば「このリンク」をクリックすると、今読んでいるページの「ナビゲーション ペイン」のセクションに移動することができます。さらに、docref と呼ばれる特別な目的を持つリンクからは、『API リファレンス』に直接アクセスすることができます。これらリンクは、たとえばこの HBox 項目へのリンクのように濃い青色の等幅フォントで表示されます。ハイパーテキスト リンクはこのような書式だけにとどまらず、グラフィック画像を含め任意の外観を持つことができます。ポインタがハイパーリンクの上に置かれると、指さし手形のマーク 
に変わります。
ドキュメント ビューアは以下の方法で起動できます。 [スタート] メニューから [ドキュメント] を選択します。Curl IDE の [ヘルプ] メニューで [言語ドキュメンテーション]、[Curl IDE のヘルプ] または [エディタのヘルプ] を選択します。Curl IDE で F1 キーを押します。
ドキュメント ビューアのウィンドウは 2 つの主な領域に分かれており、左側にはナビゲーション ペイン、右側にはコンテンツ ペインがあります。
ナビゲーション ペイン
ナビゲーション ペインは、[目次] タブ、[インデックス] タブ、[検索] タブ、[ブックマーク] タブ に分かれます。各タブでは、それぞれ異なる方法で ドキュメントから情報を検索します。
検索範囲の制御
[インデックス] タブと [検索] タブには、[検索範囲] のドロップダウン リストがあります。このリストを使用して、表示される情報を制限できます。選択肢は次のとおりです。[全ドキュメント]
開発者ガイドとユーザー ガイド、および API リファレンス、すべての IDE ドキュメントから情報を表示します。[API リファレンス]
IDE API リファレンス ドキュメントだけを検索します。[カテゴリー別の API リファレンス]
検索対象をさらに絞り込み、次のいずれかの種類のオブジェクトについてのみ、IDE API リファレンス のドキュメントを検索します。プロシージャとクラス プロシージャ非ローカル オプション定義クラスマクロとプリミティブテキスト フォーマットとテキスト プロシージャ
次の 3 つの選択肢は、指定したタイトルの Curl ドキュメントに絞り込みます。 [Curl 開発者ガイド][Curl IDE ユーザー ガイド][Curl VLE ユーザー ガイド] [目次] タブ
目次を表示するには [目次] タブを選択します。
この目次はツリー ビュー構成になっています。サブエントリのある項目は、エントリのわきに青の矢印マーク （
） が表示されます。矢印をクリックするとサブエントリが表示されます。
目次で青色の下線付き項目をクリックすると、その項目の内容がドキュメント ビューアに表示されます。クリックした項目が別の章にある場合、その章がロードされて表示されます。ページにジャンプしないトップレベルのドキュメント セクションは黒字のテキストで表示されています。
項目名がペインの幅より長くて完全に見えない場合があります。項目の上にマウス ポインタを移動すると、完全なテキストがポップアップ表示されます。[インデックス] タブ 
[インデックス] タブを選択するとドキュメントの索引が表示されます。検索範囲を絞り込んだ場合は、選択したドキュメントの索引項目だけが表示されます。
索引項目はアルファベット順に表示されています。索引の入力フィールドに入力すると、入力した文字に一致する索引一覧の最初の項目にジャンプしてそれが選択されます。文字を削除すると、残った文字列に一致するように画面の位置も変わります。ナビゲーション ペインで索引項目をクリックするとその項目が選択され、ダブルクリックするとコンテンツ ペインにその内容が表示されます。
API リファレンス内の情報の索引項目の後には、そのリンク先の API オブジェクト タイプを示す用語がカッコ内に示されています。例 :索引項目「after (マクロ)」は、API マニュアルの after マクロにリンクされています。索引項目 「after マクロ」は開発者ガイドの after マクロにリンクされています。
API クラス メンバは、Class.member 形式を使用すると、それ自体が一覧表示されます。たとえば、Canvas.add メソッドは add (メソッド) および Canvas.add (メソッド) の両方で一覧表示されます。add のように、メンバが複数のクラスで定義されている場合、メンバ項目 (add (メソッド)) をダブルクリックすると、ダイアログ ボックスが表示されクラスを選択するよう求められます。
索引項目には、メインの項目と 1 つ以上の下位項目で構成されるものがあります。メインの項目部分がドキュメントの内容にリンクされていないものもあり、これらは索引リストでは淡色で表示されます。
[検索] タブ 
[検索] タブを選択すると、ドキュメントの検索機能にアクセスできます。検索範囲を絞り込んだ場合は、検索対象は選択したドキュメントだけです。
検索機能を使用して、ドキュメントを検索できます。検索範囲を絞り込んだ場合は、選択したドキュメントだけを検索します。検索する文字列を検索の入力フィールドに入力し、Enter キーを押すか、[検索] ボタンをクリックして、検索を開始します。
[全文検索] チェック ボックスをオンにすると、ドキュメント内のすべてのテキストが検索対象になります。このチェック ボックスがオフの場合は、検索リスト内の項目だけが検索対象になります。どちらの場合も、次の 2 つのチェック ボックスを使って検索対象を変更できます。
[大文字と小文字を区別] をオンにすると、大文字と小文字を区別して検索を行います。このチェック ボックスがオフの場合、検索では大文字と小文字が区別されません。[正規表現] は、検索文字列を正規表現として解釈します。「正規表現」を参照してください。
索引用語の検索を実行すると、検索文字列を含む検索リスト内のすべての項目がアルファベット順で返されます。結果に下位項目が含まれる場合、検索結果では対応するメインの項目がその上に表示され、このメインの項目が他の項目と共にアルファベット順でソートされます。
全文検索を実行する場合、検索文字列はスペースで区切られた一連の単語として解釈されます。検索文字列に指定されたすべての単語を含むドキュメント ページが検索結果として返されます。
[ブックマーク] タブ
[ブックマーク] タブを選択すると、今後の参照のためにブックマークされたページのリストを表示できます。[移動] メニュー の [このページをブックマークする] コマンドとコンテキスト ペインのコンテキスト メニューで、ブックマークのリストに追加できます。
ペイン下部の次のボタンで、リストにあるブックマークを管理することができます。
また、ブックマークをドラッグしてリスト内の位置を移動することもできます。
ブックマークは IDE のセッション間で持続します。複数のドキュメント ビューア ウィンドウすべてで、同じブックマーク リストを共有します。ある1つのウィンドウで行われた変更は、他のすべてのウィンドウに反映されます。

ドキュメト ビューア メニュー
[ファイル] メニュー
[ソースの編集] は、現在のページのソース コードをソース エディタで開きます。[新規ドキュメンテーション ビューア] のサブメニューには、インストールされているすべての Curl ドキュメンテーションが一覧表示されます。他の言語またはリリースのドキュメントをインストールすると、このリストに追加されます。サブメニューの項目をクリックすると、新しいブラウザが開いて指定ドキュメントを表示します。[印刷] または ctrl+N は、コンテンツ ペインで現在表示されているセクションの内容を印刷用にフォーマットして、プリンタの選択を求める画面を表示します。ナビゲーション ペインは印刷されません。[閉じる] または ctrl+shift+W は、ドキュメント ビューアの現在のインスタンスを終了します。
[編集] メニュー [切り取り] または ctrl+x は、テキスト フィールドで選択したテキストをシステムのクリップボードにコピーし、選択したテキストをフィールドから削除します。このコマンドは、Curl 言語で作成されたアプレットのテキスト フィールドでテキストを選択したときにだけ使用できます。[コピー] または ctrl+c は、選択したテキストをドキュメント ビューアからシステムのクリップボードにコピーします。通常、ページに表示されるテキストはどれでも選択できます。[貼り付け] または ctrl+v は、テキストをシステムのクリップボードから Curl アプレットのテキスト フィールドに挿入します。[すべて選択] または ctrl+a は、コンテンツ ペインのコンテンツをすべて選択します。[検索] または ctrl+f は、指定したテキストの検索用ダイアログ ボックスを現在のページに表示します。
[ツール] メニュー 
[再ロード] と [停止] を除き、このメニューのすべての項目は API ドキュメントを参照する場合のみ使用できます。「API リファレンス マニュアル」を参照してください。[再ロード] はドキュメント ビューアで現在のページを再度読み込んで表示します。[停止] はページのロードを停止します。[メンバを隠す] はすべてのセクションを閉じ、メンバ情報が表示されないようにします。[メンバを表示] はすべてのセクションを開き、すべてのメンバが一覧表示されるようにします。各メンバの詳細な説明部分は非表示のままです。詳細を表示する[メンバの説明を表示] を選択後に、このメニュー コマンドを選択すると一覧表示されたすべてのメンバの詳細説明が閉じます。[メンバの説明を表示] は、一覧表示されたすべてのメンバの詳細説明を開きます。[クラスの使用者向けの表示] は既存のクラスを使いたい場合に、もっとも便利な形式でクラスのドキュメントを表示します。アクセス属性が protected のメンバは表示せず、メンバは以下のように分類されます。[プロパティ] には、アクセッサ と、ローカル オプション、非ローカル オプション が含まれます。[コンストラクタ] には、コンストラクタ と ファクトリ が含まれます。[サブクラスの作成者向けの表示] は、新しいサブクラスを作成したい場合に、もっとも便利な形式でクラスのドキュメントを表示します。それぞれの型のクラス メンバを個別の見出しの下に表示します。protected のメンバも表示します。[継承メンバ] は、クラスが継承したメンバが含まれているかどうか、および含まれている場合はそれらをグループ化する方法を選択できるサブメニューを表示します。サブメニューには次の項目があります。[なし] は、一覧表示するメンバをこのクラスで定義されたメンバに制限します。このため、継承されたメンバはリストに表示されません。[アルファベット順にソート] は、このクラスで定義されたメンバに加え、継承されたメンバも一覧表示し、アルファベット順にソートします。[クラスでソート] は、このクラスで定義されたメンバに加え、継承されたメンバも一覧表示し、クラスの継承順にソートします。[移動] メニュー 
開発者ガイドの参照中、ドキュメント ビューアはその履歴を保存していきます。[移動] メニュー を使用し、既に参照したセクションに移動することができます。[進む] または alt+rightキー は、履歴リストを一度に 1 セクションずつ前方に移動します。履歴リストからページを表示している場合は、[進む] をクリックしてドキュメント履歴内を移動し、その次に参照したセクションを表示することができます。[進む] ボタンは、履歴リストで現在のセクションの後にセクションがある場合のみ有効になります。[戻る] または alt+leftキー は、履歴リストを一度に 1 セクションずつ後方に移動します。[戻る] ボタンを押すと、現在の表示ページの前のページがロードされます。[戻る] を 複数回クリックすると、履歴内のさらに前のページが表示されます。[戻る] ボタンは、履歴リストで現在のセクションの前にセクションがある場合のみ有効になります。[このページをブックマークする] は、現在表示されているページをブックマーク リストに追加します。このリストは [ブックマーク] タブに表示されます。[履歴] は履歴リストの最後の 10 項目を表示します。履歴リストでページに移動するにはその項目をクリックします。[履歴] ウィンドウ内のページに移動するには、ページを選択して [OK] をクリックします。[履歴リスト] は [履歴] コマンドにより表示され、参照したすべてのページを一覧表示します。[ヘルプ] メニュー 
このメニューからは Curl IDE の [ヘルプ] メニューと同じ選択内容にアクセスできます。『Curl IDE ユーザー ガイド』の「ヘルプ メニュー」を参照してください。

Curl のドキュメント  
『API リファレンス』およびユーザー ガイドは、Curl 言語、Curl IDE、および関連ツールのドキュメントで構成されています。ユーザー ガイドの構成は次のとおりです。
『Curl 開発者ガイド』は、Curl 言語のハイレベルなリファレンス ドキュメントです。                                                                    『Curl IDE ユーザー ガイド』では、Curl 統合開発環境の各種ツールについて詳しく説明しています。これらのツールにより、Curl 言語でのアプレットの作成、デバッグ、パッケージ、ディプロイメントが容易になります。
The Curl VLE ユーザーガイドはCurl ビジュアル レイアウト エディター の様々なツールのドキュメントで構成されています。
API リファレンス 
『API リファレンス』では、Curl 言語の基礎と、スタンダード パッケージで利用可能な、すべてのクラスについて解説しています。クラス、プロシージャ、キーワードについて、『開発者ガイド』よりもさらに詳細に説明しています。『開発者ガイド』と同様に、『API リファレンス』にはインタラクティブな例が豊富に含まれています。
『API リファレンス』を表示するには、いくつかの方法があります。 
[目次] タブを選択して、目次から [API リファレンス] 項目を展開します。Curl 言語の全パッケージが一覧表示されます。パッケージ名を選択し、パッケージ ドキュメントを表示します。[インデックス] タブを選択して索引を表示します。ここには Curl 言語 API の主なコンポーネントが一覧表示されています。『API リファレンス』の索引項目の場合、たとえば「Box (クラス)」のように、項目名の後に API コンポーネントのタイプをカッコ内に示しています。[検索] タブを選択し、API コンポーネントを検索します。索引または検索リストで API リファレンス項目をダブルクリックします。索引および検索リストの詳細は、「ナビゲーション ペイン」を参照してください。濃い青色の等幅フォントで表示されたリンクをクリックすると、『API リファレンス』の該当項目にジャンプします。たとえば、このリンク HBox をクリックすると、HBox クラスのドキュメントが表示されます。API リファレンスには他のリファレンス項目へのリンクも数多く含まれています。
