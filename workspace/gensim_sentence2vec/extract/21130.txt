CMSという言葉を聞いたことありますか? CMSとはHTMLやCSSのような、ウェブサイト作成のための知識がなくてもサイトを作成、管理ができるシステムのことです。あらかじめ定義されたルールに従ってサイトのページが作成されるので、複数人でサイトを管理するときなど、とても便利です。
CMSを導入することにより、SEO効果もあります。CMSで作成されたウェブサイトには、SEO対策にはかかせないメタタグが自動で挿入されます。ディレクトリも管理されるため、検索エンジンからみて優良なサイトと評価され、SEO対策になります。
CMSでは、WordPress (ワードプレス)と Movable Type (ムーバブル・タイプ)が特に有名です。ここでは、WordPress と Movable Type を比較してみましょう。
WordPress の課題について
WordPress(ワードプレス)は商用でも無料で利用することができます。
開発言語はPHP。コンテンツの生成は、動的ページ生成のみとなっています。
WordPressのフリーテーマディレクトリには、千数百の無料のブログ用テーマや、ブログ機能つきのウェブサイト用テーマが登録されています。インポートするデータの形式は、「Movable Type と TypePad」「RSS」「LiveJournal」「WordPress」などから選択することができます。
ただし、WordPressは、このようなメリットばかりではありません。課題もあります。「ページの表示が遅い」「セキュリティーがあまいという声もある」「デフォルトの状態ではそれほど高いSEO効果があるわけではない」「一回のインストールで複数のブログを運営することができない」などです。
これらはプラグインである程度解決できますので、対処法を考えれば、便利なCMSになるでしょう。
Movable Type の課題について
Movable Type(ムーバブル・タイプ)は、個人事業、法人、団体の利用は有料です。
開発言語はPerl。動的ページ生成、静的ページ生成、両方できます。静的ページを生成する場合、再構築作業が必要です。
Movable Typeの課題は、ユーザーや関係者を軽視する体質という点です。たとえば、無料で利用していたユーザーから強制的にお金を徴収したり、Movable Type の普及に貢献しているはずのパートナー企業を軽視したりしていた、という点です。ユーザーの意見を聞き入れないため、ユーザーにとってやさしくないという結果になってしまっています。
