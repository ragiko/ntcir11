配置の要件に複数の言語やカルチャのサポートが含まれる場合、このトピックの情報を利用すると、ツールを選択したり、複数の言語のレポートをサポートする方法でレポートを作成することができます。また、実行時に言語リソースがどのように使用されるのかも理解できます。このトピックでは、サポートされている言語、実行時の言語リソースの使用方法、レポートの言語設定、およびローカライズされていないアイテムについて説明します。レポート マネージャー、レポート ツール バーなどのサーバー側のコンポーネントについては、言語リソースが実行時に判断されます。使用する言語リソースを決定するのは、クライアント アプリケーション (通常はブラウザー) と、レポート サーバーを実行するコンピューターのオペレーティング システムの両方です。
SQL Server 2008 での言語サポートの詳細については、「SQL Server 2008 R2 のインストールに必要なハードウェアおよびソフトウェア」を参照してください。このトピックでは、異なるタイム ゾーンにまたがってレポート サーバーにアクセスする場合の影響については説明しません。タイム ゾーンの問題の詳細については、「タイム ゾーンと時刻の設定の変更 (レポート サーバー)」を参照してください。
レポート サーバーの言語とは、レポート サーバーがインストールされているオペレーティング システムの言語です。この言語は、コンピューターの地域設定によって決まります。
レポート サーバーの言語とは、レポート サーバー インスタンスで次のアイテムに対して使用される言語です。サーバーによって返されるエラー、警告、および情報メッセージ。レポート内の書式設定 (デザイン時にレポート定義の Language プロパティが別の形で設定されていない場合)。個人用レポート機能のために作成され管理者に表示される [個人用レポート] と [Users フォルダー] のフォルダー名。個人用レポート フォルダーは、クライアントの言語設定に合わせてローカライズされます。定義済みのロールと説明。定義済みのロールには、ブラウザー、コンテンツ マネージャー、パブリッシャー、レポート ビルダー、個人用レポート、システム ユーザー、およびシステム管理者が含まれます。定義済みのロールの名前と説明は、レポート マネージャーや、レポート サーバーの権限を設定するために使用できるその他のクライアント ツールに表示されます。これらの名前はインストール時に決まりますが、インストール後に Reporting Services 構成ツールを使用して別の言語で新しいレポート サーバー データベースを作成できます。現在、SQL Server 言語は 10 個あります。特定のレポート サーバー インスタンスのレポート サーバーの言語は、常にこの 10 個の言語のいずれかになります。インストールされた 1 つの Reporting Services でこの 10 個の SQL Server 言語のすべてがサポートされます。言語パックをインストールする必要はありません。レポート サーバーの言語は、レポート サーバー データベースの作成時に設定されます。Reporting Services 構成ツールを使用してデータベースを作成する場合は、データベースの変更ウィザードを実行するときにレポート サーバーの言語を指定します。
Reporting Services を既定の構成でインストールすると、セットアップでレポート サーバー データベースが作成されるときにレポート サーバーの言語が決定されます。セットアップでは、オペレーティング システムの言語に最も近い言語リソースが使用されます。オペレーティング システムの言語と完全に一致する SQL Server 言語があった場合は、その言語が使用されます。たとえば、オペレーティング システムの言語が英語 (U.S.) の場合は、英語 (U.S.) の SQL Server 言語リソースがレポート サーバーの言語として使用されます。完全に一致する言語が見つからない場合は、最も近い言語が使用されます。たとえば、オペレーティング システムの言語が英語 (U.K.) の場合、英語 (U.K) と英語 (U.S.) は同じ言語ファミリであるため、英語 (U.K) に最も近い SQL Server 言語リソースとして英語 (U.S.) が選択されます。近い言語がない場合は、レポート サーバーの言語は英語 (U.S.) になります。
