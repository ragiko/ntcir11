	Client Agent導入済みマシンでは、ジョブ実行前/実行後に指定したプログラムを起動するような	設定が可能です。[実行遅延]を設定することで、実行前プログラムの起動からジョブが開始される	まで、又はジョブが完了してから実行後プログラムが起動されるまでの時間を秒単位で指定でき	ます。この項目で設定可能な値は0〜9999までとなっておりますが、実際に有効な値は0〜900
まで(最大15分)となっています。
［実行遅延］の設定
Client Agent 導入済みマシンで [スタート] -> [プログラム] -> [Computer Associates] -> [BrightStor] - > [ARCserve Backup Agents] -> [Backup Agent 管理] を選択 
メニューから [オプション] -> [環境設定] を選択
[実行前/実行後のプログラム] で [実行前]または[実行後]に起動コマンドを設定した後に [実行遅延] へ数値を設定
[実行遅延] に対して900秒(15分)を超える値を設定した場合、ARCserve サーバと Client Agent との間に何も通信が行なわれていないことを検知した OS が、両者間の接続を自動的に切断します。その結果ジョブが正しく動作しなかったり、指定したプログラムが実行されない現象が発生します。以下のコマンドを実行する事で OS 側からの自動切断機能を解除する事が可能となりますが、この設定を行なっても900秒以上の実行遅延は正しく動作しません。
本現象は、現段階では Client Agent の制限事項となっております。本機能をご利用の際には設定を900秒以下にして運用してください。
