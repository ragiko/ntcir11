
Windows Server 2012 と Windows 8 では、認証と承認の相関メカニズムをとおして、システム リソースとネットワーク リソースの使用を制御するのに役立ちます。ユーザーが認証された後、Windows Server 2012 と Windows 8 は Windows の承認とアクセス制御の技術を使用し、リソース保護の 2 つ目の段階である「認証されたユーザーにリソースにアクセスするための正しいアクセス許可があるかどうかの確認」を行います。
共有リソースはリソースの所有者以外のユーザーとグループにも提供されますが、未承認の使用がなされないように保護する必要があります。Windows Server 2012 と Windows 8 のアクセス制御モデルでは、ユーザーとグループ (セキュリティ プリンシパルとも呼ばれる) は一意のセキュリティ識別子 (SID) で表されます。各ユーザーとグループには、それぞれが行うことができることをオペレーティング システムに伝達する権利とアクセス許可が与えられます。各リソースには、セキュリティ プリンシパルにアクセス許可を与える所有者がいます。アクセス制御チェックでこれらのアクセス許可が調べられ、リソースにアクセスできるセキュリティ プリンシパルとそのアクセス方法が確認されます。
セキュリティ プリンシパルは、オブジェクトに対してアクション (読み取り、書き込み、変更、フル コントロールなど) を実行します。オブジェクトには、ファイル、フォルダー、プリンター、レジストリ キー、Active Directory ドメイン サービス (AD DS) オブジェクトなどがあります。共有リソースは、アクセス許可を割り当てるためのアクセス制御リスト (ACL) を使用します。リソース マネージャーは、ACL を利用し、次の 2 つの方法でアクセス制御を適用できます。
承認されていないユーザーとグループにアクセスを許可しない
承認されたユーザーとグループに与えられるアクセス権に、適切に定義された制限を設定する
オブジェクト所有者は、通常、個々のユーザーではなくセキュリティ グループにアクセス許可を与えます。既存のグループに追加されるユーザーとコンピューターには、そのグループのアクセス許可が与えられます。オブジェクト (フォルダーなど) が他のオブジェクト (サブフォルダーやファイルなど) を含むことができる場合、そのオブジェクトはコンテナーと呼ばれます。オブジェクトの階層におけるコンテナーとそのコンテンツの関係は、コンテナーを親と呼び、親のアクセス制御設定を継承する、コンテナー内のオブジェクトを子と呼ぶことによって表現されます。オブジェクトの所有者は、アクセス制御の管理を簡易化するために、個々の子オブジェクトではなく、コンテナー オブジェクトのアクセス許可を定義することがよくあります。
Windows Server 2012 と Windows 8 を使用する管理者は、オブジェクトとサブジェクトに対するアクセス制御のアプリケーションと管理を調整して次のセキュリティを提供できます。
多数の各種ネットワーク リソースが誤用されないように保護する。
組織のポリシーや業務要件と一貫性を保った方法で、ユーザーにアクセス リソースを提供する。
ユーザーが多数の場所にある多種多様なデバイスからリソースにアクセスできるようにする。
機能を更新して、ユーザーが組織のポリシーの変更やユーザーの業務の変更に合わせて定期的にリソースにアクセスしたり、増え続ける使用シナリオに対応できるようにする。使用シナリオには、たとえば遠隔地からのアクセスや、急激に拡大しているさまざまなデバイス (タブレット コンピューターや携帯電話など) からのアクセスがあります。
正規のユーザーが業務に必要なリソースにアクセスできない場合に、アクセス問題を特定し、解決する。
