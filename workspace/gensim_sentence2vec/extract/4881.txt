
プロジェクト情報のエクスポート
プロジェクトをOracle Application Server TopLink Foundation Libraryで使用するには、配布XMLを生成するか、プロジェクトをJavaソース・コードにエクスポートする必要があります。OracleAS TopLink Mapping Workbenchでは、次のプロジェクト情報の生成およびエクスポートが可能です。
プロジェクトJavaソースへのエクスポート
配布XMLのエクスポート
表作成ファイルのエクスポート
Javaモデル・ソースのエクスポート
注意: Javaソースおよび配布XMLをエクスポートすると、OracleAS TopLink Mapping WorkbenchによりJCE暗号化を使用して(JDK 1.4を使用する場合)パスワードが書き込まれます。1.3より前のJDKでパスワードの暗号化を使用する場合は、『Oracle Application Server TopLinkスタート・ガイド』を参照してください。
OracleAS TopLink Sessions Editorを使用して、カスタム・パスワード暗号化を指定します(「ログイン・プロパティの設定」を参照)。
プロジェクトJavaソースへのエクスポート
プロジェクトをJavaコードに変換するには、次の手順を実行します。通常、この生成されたコードは、XMLファイルよりも高速に実行され、簡単にデプロイできます。
プロジェクトをJavaソース・コードにエクスポートするには: 
「ナビゲータ」ペインでプロジェクトを右クリックし、ポップアップ・メニューから「エクスポート」「プロジェクトJavaソース」を選択します。
expjava.gifの説明
また、Javaソースにエクスポート・ボタンをクリックするか、またはメニューから「ファイル」「エクスポート」「Javaソースにエクスポート」、または「選択」「エクスポート」「Javaソースにエクスポート」を選択することもできます。
デプロイおよびソース・コード生成のデフォルトを定義していない場合(「プロジェクトのオプションの使用」を参照)、OracleAS TopLink Mapping Workbenchで、プロジェクト・クラス名およびソース・ディレクトリを入力するように要求されます。
