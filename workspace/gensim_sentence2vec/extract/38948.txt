
振幅プロットでは、サイン成分の実際の振幅を見ることが出来ます。 デシベル正規化プロットでは、最大ピークは 0dB、-3dB は半分のパワー、-6dB は半分の振幅を持ちます。 与えられた周波数の PSD TISA (time-integral squared Amplitude) はその周波数での (分解された) 振幅の二乗で定義されるカーブの実際の時間積分です。
正規化選択ボックスはウィンドウ適用後の二つのオプションを提供します。振幅を選択すると、使用するウィンドウのすべての値の合計を値の数で除して正規化します。ウィンドウを適用することで発生する振幅の減衰を補償することができます。スペクトルのピーク測定に有用です。パワーを選択すると、ウィンドウ適用前後のデータの平方和の比を正規化の因子とし、トータルパワーの損失が補償されます。スペクトルのトータルパワーはウィンドウ適用前のデータのパワーに相当します。
ウィンドウ
FlexPro は スペクトルの漏れ を最小化する様々な データテーパリングウィンドウ を提供します。 調整 フィールドは調整可能ウィンドウのパラメータの設定に使用します。 固定ウィンドウではこのフィールドは使用できません。
このプロシージャは通常スペクトルの漏れを削減し、時間での分解能を良くするために使用されます。 有意なオーバーラップから起こる高い余剰はそれぞれのテーパの境界での情報の損失を補います。
データウィンドウが使用されると、そのパワーがウィンドウをかけない場合と等価となるようにそれぞれのセグメントのスペクトルは正規化されます。
セグメント
それぞれのセグメントの長さ、セグメント長、とオーバーラップの総量、オーバーラップ %、は良い STFT スペクトル を得るのに重要です。 高い割合の余剰が一般に推奨されます。 セグメントサイズは、時間での分解能を最大にするためには出来る限り小さく、周波数での分解能を最大にするには出来るだけ大きくします。 時間-周波数分解能トレードオフは STFT で使用する場合は目に見えてはっきりしています。 デフォルトの 0 はセグメント長をデータ長の平方根の2倍の次の2の累乗の数に切り上げて設定します。 デフォルトのオーバーラップ 50% は周波数を時間値のおよそ倍にします。
ゼロパッディングに対応するため、FFT の実際の N、FFT 長、も指定します。 最初は FFT 長 はセグメント長と同じにするため 0 に設定されています。 ゼロパッドをするためにセグメント長よりも大きな値を入力してください。 データテーパリングウィンドウを使用すると、ゼロパッディングからごく僅かなスペクトルの漏れが発生します。 ゼロパッディングは、このアルゴリズムを使った小さなセグメントサイズを使用したときに起こる分解能の損失を与えられたピーク周波数の補間に特に有用です。
結果
一般的な STFT コンタープロットは周波数を水平方向の変数として持ち、 CWT コンタープロットは一般に時間を水平方向の変数として持ちます。 STFT と CWT を比較可能にするため、それぞれのプロットのオリエンテーションが提供されています。
オプション
最大 dB レンジ フィールドは dB または dB 正規化 フォーマットを使用している場合のみアクティブです。 dB 限界は与えられる正確な Y 勾配を指定するとともにグラフの Y レンジの自動スケーリングのために限界を与えるために使用されます。 無制限の dB レンジを指定するには 0 を入力してください。
最大 STFT 周波数は平均による縮小のための閾値を設定するために指定します。 この縮小は3次元レンダリングエンジンが管理できるサイズを保つために行われます。 スペクトルピークの減衰は起こりますが、縮小されたスペクトルは統合されたパワーを保ちます。 最大周波数値 10,000 は FFT 長 20,000 -1 まで縮小の負担を負うことがありません。 STFT グリッドのサイズはセグメントの周波数の x 数です。 周波数の数を減少したくない場合はこのフィールドに 0 を入力してください。
メモリー問題
STFT では個々の FFT はそれぞれのセグメントで生成されます。 そのため、STFT は使用可能なメモリを使い切る傾向があります。 ゼロパッディングはメモリにストアされるマグニチュードを持つ追加のスペクトル周波数に帰着します。 While FlexPro は指定した周波数カウントまで間引きを行いますが、レンダリンググリッドはそれでも大きく、システムメモリを消費します。 これが起こると、Windowsはメモリ操作のためにハードディスクを使用します。 フィジカルメモリが不十分な場合、過度のディスクスワップや極端に遅い処理、予期せぬプロシージャの終了が起こります。
STFT 実行時にメモリソースを保全するには、特別に必要な場合のみセグメントをゼロパッドし、それも周波数分解能にとって必要であるという程度までにします。 さらに、セグメントの数を増やす結果となる高いオーバーラップの設定も避けます。 オーバーラップとの関係は線形ではありません。 オーバーラップの小幅な増加はセグメントの数を増やし、要求されるメモリの総量も増えます。 FlexPro では最大 90% までのオーバーラップが可能ですが、通常 50 ～ 70% を超えても得られるものはほとんどありません。
連続ウェーブレット変換 (CWT) スペクトル
連続ウェーブレット変換 (CWT) スペクトル オプションは限られた3次元スペクトルフォーマットでの 連続ウェーブレット変換 (CWT) の表示を可能にします。 CWT は非定常として認知されているデータの探求に有用な 多分解能 時間-周波数 テクニークです。 それは、多くのスペクトルプロシージャが求めるように、データが支障なく広義に定常的であると判断されるかどうか決定することに価値があります。
