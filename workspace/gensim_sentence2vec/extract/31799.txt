セマンティック Web は、共通の柔軟な標準である RDF (Resource Description Framework) を使うことでデータの共有と統合を図れるという、新しい時代を約束しています。RDF を利用することで、さまざまなソースからのデータをマージしたり、そうしたデータに対してクエリーを行ったりすることが容易になります。しかし大量のデータが、XML やリレーショナル・データベース、そして LDAP ディレクトリーなど、RDF 以外のフォーマットで存在しています。RDF は非常に柔軟なため、こうした他のフォーマットのどれを表現することもできます。しかし既存のデータを RDF に変換する作業は膨大でコストが高く、しかも多くの場合、そうした作業は必要ありません。幸い、既存のデータを SPARQL (セマンティック Web 用のクエリー言語) を使ってクエリー可能な Web エンドポイントとして公開するためのユーティリティーがいくつかあります。そうしたユーティリティーの 1 つが、Jena Semantic Web Framework の一部である、オープンソースのユーティリティー、SquirrelRDF です (「参考文献」にリンクがあります)。この記事の目的は、LDAP ディレクトリーのための、SPARQL でクエリー可能なエンドポイントを作成するためのプロセスを説明し、またセマンティック Web に関する重要な概念を紹介することです。エンドポイントの設定方法を説明した後、Jena の Java™ クラスのいくつかの使い方を説明します。これらのクラスを利用することで、ブラウザー・ベースのクライアントから JavaScript を使ってエンドポイントのクエリーを行う際に、エンドポイントをより使いやすくすることができます。セマンティック Web とは何かセマンティック Web は新興の技術領域であり、データを表現し、データにクエリーを実行し、そしてデータにルールを適用するための一連の標準に基づいています。その中核となる標準として、表現のための RDF、クエリーのための SPARQL、構造のための RDFS、そして構造と推論のための OWL があります。セマンティック Web によってデータの統合が容易になり、また、より正確な検索、より良いナレッジ管理が可能になるなど、セマンティック Web には非常に大きな魅力があります。そのため、セマンティック Web という用語は非常に多くの意味を含むようになっています (セマンティック Web に関する標準については「参考文献」を参照してください)。RDF はセマンティック Web を構築するための基礎であり、またラベル付きの有向グラフとしてデータを表現するための標準です。資源 (resource) は、グローバルに固有で解決可能な URI でラベル付けされたエンティティーです。グラフのノードは資源とリテラルであり、矢印の付いた線で接続されます (矢印には述語のラベルが付けられます)。このグラフをシリアライズし、グラフの中のすべての矢印をリストすることができます。それぞれの矢印は、文 (statement) と呼ばれ、主語 (subject: 矢印の根もと) と述語 (predicate: 矢印のラベル)、そして目的語 (object: 矢印の向かう先) を持っています。すべての文は主語と述語、目的語を持つため、文はトリプル (triple) とも呼ばれます。すべての文の主語は、資源である必要があります。述語も資源です。文の目的語は、資源であることもあり、リテラルであることもあります。RDF と XML の間には、いくつか重要な違いがあります。まず何よりも、RDF はグラフをベースにしていますが、XML はツリーをベースにしています。RDF では矢印によって 1 つのセットが構成されるため、明示的な順序はありませんが、XML 要素には明確な順序があります。最後に、RDF は標準的なシリアライズ方法を含まないデータ・モデルであり、RDF-XML や n3、Turtle (Terse RDF Triple Language: 「参考文献」を参照) など、いくつかのフォーマットでシリアライズすることができます。リスト 1 の例は 2 人の人間の連絡先情報をエンコードするものですが、Turtle で作成されています。リスト 1.
