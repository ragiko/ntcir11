画像の表示
このセクションでは、画像の作成時や既存の画像のスクリプトへの挿入時に使用するImage Viewer(画像ビューア)と関連パネルを説明します。
Image Viewer(画像ビューア)
画像は、左側のImages(画像)パネルへ進むと、Suiteウィンドウで表示できます。
画像のプレビュー
一度クリックされた画像は、"プレビュー"モードで表示されます。Suiteウィンドウの左側のImages(画像)パネルで画像をクリックすると、画像を閲覧できます。画像をダブルクリックするか、プレビュー表示の中でクリックすると、パーマネントタブに上げられます。
Image Well: キャプチャ画像を現在保存されている通りに表示します。
Image Name Field(画像名)フィールド: このフィールドでは、画像の名前を編集できます。
Hot Spot: 画像の保存後、画像の左上からの相対位置の座標を調整すると、ホットスポットを移動することができます。
Image Doctor: Image Doctorエリアでは、いつでも(利用できる有効なリモート接続で)画像の検索が可能です(このエリアの左下のSearch(検索)ボタンを使用)。ここではどのような設定が画像の検出に最適であるかを確認するため、診断的なさまざまな検索法を使用します。検索の例を以下に挙げます。
Standard Search:調整なしの通常の画像の状態です。
Dynamic Tolerance:この検索は画像許容差の設定を調節します。画像がViewer(ビューア)ウィンドウで検出されるまで、画像の通常の許容差設定で開始し、段階的に許容差を上げます。
Alternate Types:この検索は、一致がみられるまで徐々にゆるやかな検索タイプを適用します。
Standard Search:この検索は、一致がみられるまで0.5～2.0の範囲のさまざまなスケールで画像を調査します。 
Discrepancy Search:この検索は、画像の不一致設定を一致がみられるまで段階的に上げていきます。
Original Location:この検索は、最初の取込み時に選択された画像の位置を示します。
Searching: 検索タイプは、eggPlantがViewer(ビューア)ウィンドウの一部をターゲット画像の一致部分とみなすようにeggPlantが定める精度のレベルを記述したものです。画像ビューアのこのエリアでは、画像検索に関連した以下の設定を変更できます。
Precise: Precise(精度)検索タイプは、Viewer(ビューア)ウィンドウで一致した画像とみなす場合の精度を非常に高度に定めます。これは、わずかな色差や微妙なコントラストのある画像に役立ちます。
Tolerant: Tolerant(許容差)検索タイプは、背景や透明度の変更により生じるわずかに異なるピクセルの色をもつ一致を考慮します。Tolerant(許容差)検索は、精度の厳密な検索よりも許容範囲が広いため、レンダリングのわずかな違いが原因でスクリプトが失敗する危険を抑えます。
Pulsing: Pulsing(パルス化)検索のタイプは、Mac OS XのPulsing(パルス化)ボタンなど、変更可能なピクセルの色を考慮します。
Text: Text(テキスト)検索タイプは、テキストを含む画像の動的なフォントスムージング(テキストアンチエイリアス)を考慮します。eggPlantがテキストを含む画像の位置づけに時々失敗する場合は、検索タイプをText(テキスト)に変更してみてください。
Text&Pulsing: Text&Pulsing(テキスト&パルス化)検索タイプは、ピクセルの色の変更とテキストアンチエイリアスの両者を考慮します。 
Image Description: このフィールドは画像の説明用です。オプション。
Image Browser: このブラウザでは、画像をナビゲートし、すぐに多くのまたはすべての画像を閲覧できます。こちらをただクリックするだけで画像が開きます。Tip:このセクションでは、WindowsのCtrl Click(コントロールクリック)、MacのCmd Clickを行うことで、すぐに複数の画像を選択できます。その後、全体として検索タイプなどの設定を変更します。
Image Capture(画像キャプチャ)パネル
Capture(キャプチャ)パネルは、追加スクリプトや画像コントロールなどからSave(保存)ダイアログと似ています。 
Image Name Field(画像名)フィールド:Image Name(画像名)フィールドは、image0001から始まる現在のスイートに特有の動的に生成された名前にデフォルト設定されます。この名前は一時的なスクリプトや実験には十分ですが、本格的にスクリプトを開始するときには説明的な記述名の方が作業を進めやすくなります。画像の名前には、英数字、スペースのほか、ほとんどの句読記号を含めることができます。
Where Pop-Up Menu:Where (位置)のポップアップメニューは、画像を保存するフォルダを表示します。デフォルト選択では現在のスイートの画像フォルダです。
Image Well:Capture(キャプチャ)パネルの中央部分は、キャプチャ画像を表示する画像イメージです。画像内のホットスポットは、赤い十字線で表示されます。ホットスポットを移動させるには、十字線をControl(コントロール)-クリック(Mac OS XではCommand(コマンド)-クリック)するかControl(コントロール)-ドラッグ(Mac OS XではCommand (コマンド)-ドラッグ)します。注:キャプチャ画像が画像イメージには大きすぎる場合は縮小してサイズをフィットさせます。実際の画像は影響を受けませんが、Capture(キャプチャ)パネルではゆがんでみえる可能性があります。 
Search Type Pop-Up Menu:Search Type(検索タイプ)のポップアップメニューでは、画像キャプチャー時にデフォルト以外の検索タイプを選択できます。
注:デフォルトの検索タイプSearch Type(検索タイプ)のポップアップメニューのデフォルト選択は、保存している画像によって異なります。eggPlantは各画像に最適な検索タイプを選択するようにします。(Tolerant(許容差)は、通常は最も一般的なデフォルト設定の検索タイプであることから、最適な選択です。Text(テキスト)および Text&Pulsing(テキスト&パルス化)は、決してデフォルト選択となりません。)注:Run Option(実行オプション)設定では、Precise(精度)およびTolerant(許容差)の検索タイプ設定の許容範囲を調整できます。
Command/Function Pop-Up Menu:画像付きのスクリプトにコマンドまたはファンクションを挿入する場合は、Command/Function(コマンド/ファンクション)のポップアップメニューで、コマンドまたはファンクションを選択します。 コマンドまたはファンクションのないスクリプトに画像を挿入する場合は、 Capture Image(画像キャプチャ)を選択します。スクリプトの選択したコマンドまたはファンクションに画像を挿入する場合は、Add Image(画像の追加)を選択します。
Creating a New Folder: 現在選択しているフォルダ内にサブフォルダを作成する場合は、New Folder(新規フォルダ)ボタンをクリックします。
Making an Image Collection: 既存の画像に応じてImage Collection(画像コレクション)を作成するには、MakeCollection(画像の作成)ボタンをクリックします。Image Collection(画像コレクション)には画像と同じ名前が付き、画像は自動的にImage Collection(画像コレクション)に移動します。 Image Collection(画像コレクション)の作成の詳細は、こちらをクリックしてください。
Max Wait Field: Wait(待機)またはWaitFor(待機対象)と関連のある最長待機時間を指定するには、Max Wait(最長待機)フィールドに時間(秒単位)を入力してください。 
Cancel Button: 画像キャプチャをキャンセルし、Capture(キャプチャ)パネルを閉じる場合は、Cancel(キャンセル)ボタンをクリックしてください。
Save Button: 画像キャプチャを完了し、Capture(キャプチャ)パネルを閉じる場合は、Save(保存)ボタンをクリックしてください。
Use Image(画像の使用)(Mac OS XおよびLinuxのみ)
Use Image(画像の使用)パネルは、一番下にスクリプトのコントロール類が追加されており、標準的なOpen(オープン)ダイアログに似ています。Capture(キャプチャ)パネルを通常開くときのAlt-コマンドをクリック(Mac OS Xの場合はOption(オプション)-コマンドをクリック)でもUse Image(画像の使用)パネルを開くことができます。 
File Browser: Use Image(画像の使用)パネルの上部は標準のファイルブラウザです。現在のスイートのImage(画像)ディレクトリとサブフォルダ間をナビゲートするために使用します。 
Command/Function Pop-Up Menu: コマンドまたはファンクションを画像付きのスクリプトに挿入するには、Command/Function (コマンド/ファンクション)のポップアップメニューでを選択します。スクリプトの選択したコマンドまたはファンクションに画像を挿入する場合は、Add Image(画像の追加)を選択します。 
Max Wait Field: Wait(待機)またはWaitFor(待機対象)と関連のある最長待機時間を指定するには、Max Wait(最長待機)フィールドに時間(秒単位)を入力してください。 
Cancel Button: 画像の選択をキャンセルし、Use Image(画像の使用)パネルを閉じる場合は、Cancel(キャンセル)ボタンをクリックします。
Insert Button:  Insert(挿入)ボタンにより、Use Image(画像の使用)の操作が完了します。生成されたコマンドまたはファンクションをスクリプトに挿入し、コマンド(可能な場合)を実行して、Use Image(画像の使用)パネルを閉じます。
Find Text(テキストの検索)パネル 
Find Text(テキストの検索)パネルでは、キャプチャ画像のないテキストを検索できます。これにより、OCR(光学的文字認識)かTIG(テキスト画像ジェネレータ)でテキストを検索できます。
OCR: デフォルトのプラットフォームは、eggPlantで光学的文字認識の機能を利用できるGeneric OCR(一般的なOCR) です。OCRは解釈のアルゴリズムを使用し、SUTの画面でピクセルパターンを調べ、画面に存在する文字を識別します。これにより、調査結果が辞書(デフォルトの辞書は英語)と照合され、各種辞書にアクセスすることで、さまざまな言語を読み取ります。対応言語のリストについては、付録G:OCRの対応言語を参照してください。
TIG: これはSUTにインストールされたテキスト画像ジェネレータで、テキストの画像を動的に生成してから検索します。テキスト画像ジェネレータは、TestPlantのダウンロードページからダウンロードできます。
Find Text(テキストの検索)パネルを開くには、Viewer(ビューア)ウィンドウでFind Text(テキストの検索)ボタンをクリックするか、Control(コントロール)メニュー>Find Text(テキストの検索)を選択します。
Find Text(テキストの検索)パネルにはテキストプロパティが2セットあります。一つはOCR(光学的文字認識)を使用するテキストプラットフォーム用、もう一つは定義済みのテキストの画像を動的に生成するプラットフォーム用です。
Text String Field:  Text String(テキスト文字列)フィールドには、画像内に表示される実際のテキストが含まれています。テキストプロパティリストを挿入できるようにする前に、こちらにテキストを入力する必要があります。
Platform Pop-up Menu: Platform(プラットフォーム)ポップアップメニューは、テキスト設定で定義するテキストプラットフォームから成ります。テキストプラットフォームは通常、SUTのオペレーティングシステムによって識別されます。作業中のSUTを示す一般的なテキストプラットフォームまたはテキストプラットフォームから選択します。(テキストプラットフォームとTIGの詳細については、eggPlantの利用について:「テキストプラットフォームの詳細」を参照してください。)
Style Pop-Up Menu: Style(スタイル)のポップアップメニューには、所定のテキストスタイルの選択肢が含まれています。スタイルをスタートポイントとして選択した後は、必要に応じて個々の属性を変更できます。(たとえば、テキストがSUTの標準のメニューフォントを使用し赤で表示する場合、メニュースタイルを選択してからテキストカラーを赤に変更します。)Text preferences(テキストの設定)では、新しいスタイルを定義することができます。
Text Language Pop-Up Menu: Text Language(テキスト言語)のポップアップメニューには、OCRの言語辞書の選択肢が含まれています。
Case Sensitive: Case Sensitive(大小文字識別)チェックボックスでは、検索時に大文字と小文字を識別するか否かを決定します。
Contrast On: Contrast On(コントラストオン)チェックボックスでは、検索時にコントラストをオンにするか否かを決定します。これは、アンチエイリアス処理されたテキストでの作業時にしばしば役立ちます。OCRエンジンにツートーンでみえるように、SUTの画面をフィルターに通します。コントラストのTolerance(許容差)とコントラストカラーを設定できます。最適な方法:状況によってはテキストカラーを使用できますが、一般には背景色をコントラストカラーとして使用することをおすすめします。
Command/Function Pop-Up Menu: テキストプロパティリストでコマンドまたはファンクションをスクリプトに挿入するには、Command/Function(コマンド/ファンクション)のポップアップメニューから選択します。ポップアップメニューの下のプレビューフィールドは、スクリプトに挿入されるテキストを表示しています。スクリプトの選択したコマンドまたはファンクションにテキスト画像を組み込むには、Add Image(画像の追加)を選択します。
Max Wait Field: Wait(待機)またはWaitFor(待機対象)と関連のある最長待機時間を指定するには、Max Wait(最長待機)フィールドに時間(秒単位)を入力してください。 
Cancel Button: テキストプロパティリストをキャンセルし、Find Text(テキストの検索)パネルを閉じる場合は、Cancel(キャンセル)ボタンをクリックします。 
Save Button: Save(保存)ボタンは、選択したコマンドまたはファンクションで、スクリプトにテキストプロパティリスト挿入し、コマンド(必要に応じて)を実行してからFind Text(テキストの検索)パネルを選択します。 
TIGのみの要素:
フォントおよびサイズのフィールド: Font and Size(フォントおよびサイズ)のフィールドでは、テキストのフォントやサイズを選択したスタイルと異なる場合に選択します。 
太字、イタリック体、下線表示のチェックボックス: これらのチェックボックスを選択すると、テキストイメージのテキストに太字、イタリック体、または下線表示が施されます。すべてのフォントが太字またはイタリック体で描画できるわけではないため、フォントとスタイルの組合せによっては、使用できるテキスト画像が生成されないこともあります。注:下線表示を描画する方法はプログラムごとに異なるため、画像に下線が施されている場合、必要とされる正確な画像を生成できない可能性があります。そのような場合は、Capture Image(画像キャプチャ)を使用して、Viewer(ビューア)ウィンドウでテキストのスナップショットを撮ります。
テキスト画像のプレビュー: 画像プレビューエリアを表示または非表示にするには、開示三角形またはText Image Preview(テキスト画像のプレビュー)の文字をクリックします。選択したテキストプラットフォームで、eggPlantのオリジナルのテキスト画像ジェネレータ(TIG)またはネットワーク上の別の迅速なTIGを使用する場合、テキスト画像のプレビューが役立ちます。選択したテキストプラットフォームがより緩徐なTIGを使用する場合や、プラットフォームのTIGが現在利用できない場合は、画像プレビューにより、システムを遅くすることができます。
Search Type Pop-Up Menu: 検索タイプは、eggPlantがViewer(ビューア)ウィンドウの一部をターゲット画像の一致部分とみなすようにeggPlantが定める精度のレベルを記述したものです。テキストについては、最適な検索タイプは通常、Text(テキスト)です。(詳細については、上述のCapture(キャプチャ)パネルのSearch Type(検索タイプ)のポップアップメニューを参照してください。このポップアップメニューは、画像プレビューエリアが表示される場合にのみ利用できます。
テキストと背景色のイメージ:テキストまたは背景の色を編集するには、Colors(カラー)パネルでそれぞれの色をクリックし、新規の色を選択します。ヒント:Colors(カラー)パネルのカラーピッカーは、特にeggPlantで便利です。Viewer(ビューア)ウィンドウなど、ディスプレイの任意の場所からカラーをコピーできます。カラーピッカーを利用する場合は、Colors(カラー)パネルの拡大鏡をクリックし、コピーしたい色のところで再度クリックします。
