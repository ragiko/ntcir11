今回の記事は Bamboo 3.1 に関する、2011 年 5 月 10 日に発表された Giancarlo Lionetti 氏のブログ投稿「Bamboo 3.1 – Continuous Deployment…Continued」の弊社日本語訳です。原文はこちらでご確認いただけます。
83 日リリース、新しいビルド概念、パラメーター化されたビルド、Bitbucket および Git コネクタ、および改善された .Net サポートを組み合わせると何が生まれるか? これは Bamboo 3.1 と呼ばれ、本日リリースされました。
Bamboo 3.1 の新機能
1. ビルダーに代わるタスク
ビルダーはタスクと呼ばれるより柔軟性および拡張性の高い概念に置き換えられました。複数のタスクが同じビルド ディレクトリへのアクセスを持って同じジョブに連続的に実行できます。以前はジョブは 1 つのビルダー (Ant、Maven、MSBuild など) に限定されていました。Maven Goal や Ant Target の実行以上のことに関して、いくつかの操作を実行するスクリプトを作成する必要がありました。そこでタスクの登場です。
ソフトウェアをパッケージ化し、Web サイトにアップロードします。QA サーバーまたは実稼働環境に新しいバージョンを継続的に展開します。タスクはプラグインとして実装される (作成も簡単) ため、タスクで行うことは完全にユーザーに委ねられます。Atlassian プラグイン ライブラリで利用可能なタスクを見つけ、Bamboo インスタンスにインストールしてください。
既存のビルダーはすべてタスクに変換され、テスト済みです (カスタム ビルダーがある場合はアップグレードすることをお勧めします)。
2. プラン変数とパラメーター化されたビルド
ビルド構成で使用するためにグローバル変数を指定することはこれまでもできました。しかし、すべてのプランがこれらの変数を共有し、管理者だけが作成できるため、個々のプロジェクトがこの機能を本当の意味で活用することは困難でした。簡単にビルドを管理するため、弊社はプラン変数を導入しました。1 つのプランの複数のジョブ間でビルド スクリプトとタスクで使用できる変数を指定できるようになりました。
これをさらに一歩進めて、私たちは開発プロセス中にカスタマイズされたパラメーターでビルドを実行したい場合があることに気付きました。これらの “特殊ビルド” はりリースおよび展開プロセスにおいて重要な役割を果たします。Bamboo 3.1 では手動でトリガーしたときにビルドをパラメーター化でき、ビルド構成で使用したプランやグローバル変数を上書きします。
3. .NET サポート
ビルダーをタスクに作り直す際に Bamboo の .Net サポートを改善する機会があり、これを行いました。以下の新機能の実装に役立った皆様からのフィードバックに感謝しています。
Visual Studio – Visual Studio プロジェクト devenv.exe のビルド。x86、AMD64、IA32、IA64      などのさまざまなアーキテクチャーをサポート
MSBuild – ビルドの一環として      MSBuild を実行
NAnt – プロジェクトをビルドするために NAnt ターゲットを実行
MSTest パーサー – MSTest テスト結果を解析および表示
MBUnit パーサー – MBUnit テスト結果の解析および表示
NUnit パーサー – NUnit テスト結果の解析および表示
この新機能についてご意見をお寄せください。また、他に必要な改善点や .Net 機能がある場合はお知らせください。
4. Bitbucket および GitHub サポートの改善
Bamboo 3.1 では Bitbucket および GitHub の統合が改善されています。Bitbucket または GitHub レポジトリを構成するときにユーザー名とパスワードを入力するだけで、Bamboo は Bitbucket または GitHub から便利なドロップダウンで利用可能なレポジトリを読み込みます。
5. ユニバーサル プラグイン マネージャー (UPM)
プラグインの管理と Bamboo のアップグレードの実行がさらに簡単になりました。JIRA と Confluence に同梱済みのユニバーサル プラグイン マネージャー (UPM) が Bamboo にも同梱されるようになりました。UPM では以下のことを行えます。
Bamboo のアップグレード前にプラグインの互換性チェックを実行します。
コミュニティ タスクのように、Atlassian プラグイン ライブラリから新規プラグインをインストールします。
既存のプラグインを管理します。
ワンクリックで利用可能な更新プログラムがあるすべてのプラグインをアップグレードします。
監査ログを使用して更新を表示および追跡します。
その他の利点
Amazon EC2 Spot Instances のサポート : 未使用の Amazon EC2 容量を入札し、より低コストでビルドを実行できます。
Gravatar のサポート : ビルドを壊したユーザーの写真を表示します。
詳細については、Bamboo 3.1 リリース ノートを参照してください。
