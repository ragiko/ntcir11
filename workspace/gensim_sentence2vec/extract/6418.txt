
方針:
全保管域タイプに厳密な FIFO 
を適用 
方針の説明
一般に、保管域タイプごとに 1 
つのピッキング方針が割り当てられているものと想定されています。ただし、保管域タイプにもとづく方針は、すべての業態で使用できるわけではありません。
倉庫管理システムでは、
倉庫内のすべての保管ロットを考慮に入れることにより、FIFO
方針に従って在庫をピッキングすることができます。この場合は、各保管域タイプが順番に検索されるのではなく、倉庫全体で最も古い保管ロットが、倉庫から払い出される保管ロットとして提示されます。保管域タイプ別に指定されたピッキング方針は考慮されません。このため、不整合が生じることがあります。
したがって、不適正なピッキングが行われないように、保管域タイプの一部を厳密なFIFO
方針から除外する必要があります。たとえば、差異仮保管域は、厳密な FIFO
方針から除外する必要があります。厳密な FIFO
方針は、テーブル厳密な FIFO
からの保管域タイプの除外
で設定または設定解除します (
以下を参照)
。
方針の設定
保管域タイプ検索
テーブルで、以下に示すように最初の保管域タイプ列にアスタリスク 3
つ ( ***)
を入力します。
保管域タイプ検索テーブル
Strategy: Stringent FIFO Across All Storage Types - Warehouse Management System (WMS) - SAP Library
