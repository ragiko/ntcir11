今日学習したこと
テストの一般原則
1.テストは欠陥があることしか示せない
2.全数テストは不可能
3.初期テスト
4.欠陥の偏在
5.殺虫剤のパラドックス
6.テストは条件次第
7.「バグゼロ」の落とし穴
用語
・テスト  … 欠陥から発生する故障を見つけること
・デバッグ … 欠陥の原因を突き止め、ソースコードを修正し、
欠陥も正しく修正したことを確認する一連の
開発作業
・インシデント  … 発生した事象の中で調査が必要なもの
・テストハーネス … テスト実行に必要なスタブやドライバから成る
テスト環境
・テストスイート … テスト対象のコンポーネントやシステムのために
テストケースをまとめたもの
・テストサマリレポート
… テスト作業と結果を要約した報告用ドキュメント
・イテレーティブ−インクリメンタル開発モデル
… システムの要件、設計、構築、テストを何回かの
短い開発サイクルで繰り返してソフトウェアを
作るモデル
・RAD       … ソフトウェア開発技法の一つ。プロトタイプと
呼ばれるシステムの完成イメージを何度も制作、
評価し、プロトタイプを次第に完成品に近づけて
ゆく手法
・RUP       … IBM社ラショナルブランドのオブジェクト指向型
ソフトウェア開発プロセス、およびその製品の
ことを指す
・アジャイル開発モデル
… ソフトウェア工学において迅速かつ適応的に
ソフトウェア開発を行う軽量な開発手法群の総称である
・デプロイ    … ネットワークアプリケーションやWebサービスなどを
利用可能なように準備すること
・ロバストネステスト
… ソフトウェアの頑健性を判定するテスト
・ブランチカバレッジ
… テストスイートによって用いられた分岐のパーセンテージ
・インシデントレポート
… 発生したあらゆるインシデントを報告するドキュメント
・テスト駆動型開発… 本体のソフトウェアを開発する前に、テストケースの
開発するソフトウェア開発手法
・デシジョンテーブル
… 入力と刺激、および、対応する出力と処理の
組み合わせを示す表
・アルファテスト … 開発組織内で実施するテスト
・ベータテスト  … 外部ユーザによるテスト
・性能テスト   … ソフトウェア製品のパフォーマンスを判定するための
テスト手順
・ロードテスト  … コンポーネントやシステムの動作を測定するテストの一種
・ストレステスト … 要件で定義した限界、または、それを超えた条件で
システムやコンポーネントを評価するテスト
・ユーザビリティテスト
… ソフトウェア製品が指定された条件下で理解され、
学びやすく、使用しやすく、ユーザにとって魅力的で
ある能力を判定するためのテスト
・相互運用性テスト… ソフトウェア製品の相互運用性を判定するテスト手順
・保守性テスト  … ソフトウェア製品の保守性を判定するテスト手順
・信頼性テスト  … ソフトウェア製品の信頼性を判定するテスト手順
・移植性テスト  … ソフトウェア製品の移植性を判定するテスト手順
・ステートメントカバレッジ
… テストスイートによって実行されたステートメントの
パーセンテージ
・デシジョンカバレッジ
… テストスイートによって用いられた判定結果の
パーセンテージ
・影響度分析   … ある要件を変更する前に、開発手引書、テスト手引書、
コンポーネントの各階層が変更となり、どんな影響を
受けるか評価すること
・インスペクション… ピアレビューの一種。目視検査により欠陥を
抽出する方法
・メトリクス   … 測定尺度、および、測定方法
ここまで。
