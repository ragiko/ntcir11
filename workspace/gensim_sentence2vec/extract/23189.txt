
 SAP EP 6.0 のポータルサーバ設定の更新 
用途
EP 6.0 のポータルサーバ設定更新カスタマイジングアクティビティでは、接続済ポータルテーブル(技術名称: RSPOR_T_PORTAL) でさまざまな設定を行う必要があります。
接続済ポータルテーブルは、すべてバッファされます。このテーブルを更新したら、SAP BW アプリケーションサーバ上のテーブルバッファを初期化する必要があります。初期化は、SAP GUI の /$tab で実行できます。
これらの設定は、すべての統合シナリオに必要です(
管理者および作成者の観点による統合を参照)。
手順
...
 
1. 
トランザクションテーブルビュー更新 (トランザクションコード SM30) を開始します。
 
2. 
テーブルとして RSPOR_T_PORTAL
と入力します。
 
3. 
更新を選択します。
 
4. 
新規エントリを登録する場合は、新規エントリを選択します。
 
5. 
接続済ポータルを更新します。
 
RFC 宛先: <RFC_DESTINATION>
 
システムの名称: <SYSTEMALIAS> (ポータルでの BW システムの登録を参照)
 
デフォルト: <DEFAULT>
 
ポータルのURL プレフィックス: <PORTAL_URL_PREFIX> (例: http://<ポータルサーバ><ドメイン>:<ポート>)
 
BW メタデータの RM 接頭辞: <KM_RM_METADATA_PREFIX> (例: /bw_metadata)
 
KM サービス URL: <空白>
 
6. 
エントリを保存します。

接続済ポータルに関する設定
RFC 宛先とシステムの名称
SAP BW 3.5 とSAP EP 6.0 を統合するためには、トランザクションSM59 を使用して RFC 宛先を更新する必要があります。この RFC 宛先は接続済ポータルテーブル (RSPOR_T_PORTAL) に入力します。それによって、この RFC 宛先は接続済ポータルとして認識されます。
BW システムとSAP エンタープライズポータルの間での通信は、この RFC 宛先をもとにして行われます。各 RFC 宛先は、接続済ポータルに対応します。システムの名称項目は、BW システムのエイリアスに対応します。エイリアスは、SAP エンタープライズポータル 6.0 で定義されます。システムの名称を使用して、エンタープライズポータルの iView を BW システムへ明確に割り当てることができます。
ポータルが複数ある場合については、ポータル証明書のインポートおよびポータル証明書のエクスポートを参照してください。
インフォメーションブロードキャスティングに関する設定
接続済ポータルテーブル(RSPOR_T_PORTAL) では、RFC 宛先やシステムの名称の他にも、デフォルト、URL プレフィックス、KM Metad.RM 接頭辞、KM サービスURL の各項目を更新することができます。インフォメーションブロードキャスティングに対しては、これらの項目を更新する必要があります。BEx ブロードキャスタでは、ナレッジマネジメント内のフォルダ (KM フォルダ) で入力ヘルプが使用できるようにするため、これらの情報が必要となります (
エンタープライズポータルへのエクスポートを参照)。クエリまたは BEx Web アプリケーションへのオンラインリンクをナレッジマネジメントに格納するためには、BW メタデータリポジトリマネージャの接頭辞が必要です。これらの項目は、その他のシナリオでは必要ありません。
テーブルからポータルを 1 つだけ選択し、それを標準ポータルとして指定する必要があります。指定されたポータルは、BEx ブロードキャスタでフォルダを選択する際に使用されます。
標準ポータルは、パラメータ ID RSPOR_DEFAULT_PORTAL (SET/GET
パラメータ) を使用して変更することができます。
...
 
1. 
SAP GUI で、メニューのシステムユーザプロファイルユーザデータを選択します。
 
2. 
パラメータタブページで、パラメータID RSPOR_DEFAULT_PORTAL および適切なパラメータ値を入力します。入力するパラメータ値は、接続済ポータルテーブルにある RFC 宛先の名称です。
 
3. 
 
URL プレフィックス項目には、プロトコル、ホスト名およびポートで構成されるポータルへのURL を指定します。
KM Metad.RM 接頭辞項目で、BW メタデータリポジトリマネージャの接頭辞を更新します。この接頭辞は、リポジトリマネージャの設定中に指定されます。
KM サービスURL では、必要に応じて、ナレッジマネジメント内のフォルダの入力ヘルプに対して別のサービスを使用することができます。現時点では、サービスはただ 1 つであるため、この項目は空のままです。
テーブルを更新しないと、BEx Web アプリケーションデザイナおよび BEx クエリデザイナでの公開に関連するメニューエントリは無効化されます。
チェック
BEx Web アプリケーションデザイナを開始します。公開メニューで、エンタープライズポータル 6.0のエントリが有効化されます。
