収束パラメータを調整する
更新日: 2005年1月適用対象: Windows Server 2003, Windows Server 2003 R2, Windows Server 2003 with SP1, Windows Server 2003 with SP2
[スタート] ボタン、[ファイル名を指定して実行] の順にクリックし、[名前] に「regedit」と入力します。
次のレジストリ キーをクリックします。  
HKEY_LOCAL_MACHINE\System\CurrentControlSet
\Services\WLBS\Parameters\Interface\<ネットワーク アダプタ ID>
必要に応じて AliveMsgPeriod パラメータか AliveMsgTolerance パラメータを編集します。
コンピュータを再起動します。
注意
レジストリを正しく編集しないと、システムが正常に動作しなくなる場合があります。レジストリを変更する前に、コンピュータ上の重要なデータのバックアップを作成する必要があります。
注
この手順を実行するには、ローカル コンピューターの Administrators グループのメンバーであるか、または適切な権限が委任されている必要があります。コンピューターがドメインに参加していると、Domain Admins グループのメンバーがこの手順を実行できる場合もあります。セキュリティを考慮するうえで最適な方法として、この手順を実行するときに [別のユーザーとして実行] を使うことを検討してください。
この手順の目的は、メッセージ交換の間隔と、メッセージが何回受信されなかったら収束を開始するかの両方を制御できるようにすることです。詳細については、"関連項目" を参照してください。
AliveMsgPeriod の値はミリ秒単位でメッセージ交換の間隔を保持し、AliveMsgTolerance の値は、ホストからの交換メッセージが何回受信されなかったらクラスタが収束を開始するかを指定します。  
これらの値は、インストールの要件を満たすものでなければなりません。メッセージ交換の間隔が長いほど、フォールト トレランスを維持するために必要なハートビート トラフィックが減少します。ただし、ネットワーク負荷分散がオフライン ホストへのネットワーク メッセージの送信を停止するまでの時間は長くなります。同様に、収束までのメッセージ交換の回数を増やすと、ネットワークの一時的な混雑に起因して不要な収束が開始される回数を減らすことができますが、この場合もまた、オフライン状態になったホストがネットワーク トラフィックの受信を停止するまでの時間は長くなります。
既定値を使った場合は、ホストが見つからないと判断されるまでに 5秒を要し、クラスタが負荷を再度割り当てるのにさらに 5秒を要します。オフライン状態になったホストへのネットワーク トラフィックの送信が停止されるまでに合計 10秒の時間がかかりますが、ほとんどの TCP/IP アプリケーションでは、この程度の時間は許容されます。この構成では、ネットワークのオーバーヘッドはほとんど発生しません。
WLBS は WindowsNT Load Balancing Service の略語であり、WindowsNT4.0 でネットワーク負荷分散を表す名前として使われていました。下位互換性を保つために、場合によっては WLBS がそのまま使われています。
