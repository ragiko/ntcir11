第35回は「今週のアルゴリズム:ソートの交換回数を合計」の問題です。
問題文
アルゴリズムの基本ともいえるソート。
様々な手法が存在しますが、今回は処理時間の高速化ではなく、交換回数を最小化する方法です。
例えば、1, 2, 3の3つの数字について、2つの数字を交換することを繰り返し、小さい方から順に並べることを考えます。
最初の並び順によって、並び替えるのに必要な最小の交換回数は異なっています。
1, 2, 3  交換不要
1, 3, 2 (2と3を交換) 1, 2, 3 (1回)
2, 1, 3 (1と2を交換) 1, 2, 3 (1回)
2, 3, 1 (1と2を交換) 1, 3, 2 (2と3を交換) 1, 2, 3 (2回)
3, 1, 2 (1と3を交換) 1, 3, 2 (2と3を交換) 1, 2, 3 (2回)
3, 2, 1 (1と3を交換) 1, 2, 3 (1回)
つまり、1, 2, 3の3つの数字の場合、最小の交換回数の合計は7回になります。
では、1～7の7つの数字からできるすべての順列からはじめ、小さい方から順に並べるように最小回数で交換を行うとき、その交換回数の合計を求めてください。
