VMM における仮想ネットワークの構成
更新日: 2010年3月適用対象: Virtual Machine Manager 2008, Virtual Machine Manager 2008 R2, Virtual Machine Manager 2008 R2 SP1
Hyper-V と共に Windows Server2008 を実行しているバーチャル マシン ホスト、または Hyper-V と共に Windows Server2008R2 を実行しているバーチャル マシン ホストでは、System Center Virtual Machine Manager (VMM)2008 および VMM2008R2 は、Hyper-V の仮想ネットワークを使用します。ここでは、VMM での仮想ネットワークの構成方法について説明します。Hyper-V での仮想ネットワークの構成についての詳細は、「仮想ネットワークを構成する」(http://go.microsoft.com/fwlink/?LinkId=158767) を参照してください。
仮想ネットワークの種類
Hyper-V ホスト上で仮想ネットワークを作成して、バーチャル マシンとホストのための様々なネットワーク トポロジを定義できます。VMM では、Hyper-V ホスト上で 3 種類の異なる仮想ネットワークを作成できます。
プライベート仮想ネットワーク。同じホスト上のバーチャル マシン間で通信を行い、ホストまたは外部ネットワークとの通信は行わない場合は、この種類を使用します。プライベート仮想ネットワークは、ホスト オペレーティング システム内に仮想ネットワーク アダプターをもたず、物理ネットワーク アダプターにバインドされません。プライベート仮想ネットワークは通常、ホスト オペレーティング システムおよび外部ネットワーク内のネットワーク トラフィックからバーチャル マシンを分離する場合に使用します。
内部仮想ネットワーク。同じホスト上のバーチャル マシン間の通信、およびバーチャル マシンとホストの間で通信を行う場合は、この種類を使用します。内部仮想ネットワークは、物理ネットワーク アダプターにバインドされません。内部仮想ネットワークは通常、ホスト オペレーティング システムからバーチャル マチンに接続しなければならない場合のテスト環境を構築するために使用します。
外部仮想ネットワーク (物理ネットワーク アダプター)。各バーチャルマシン間、バーチャル マシンと外部に配置されているサーバー、およびオプションでバーチャル マシンとホスト オペレーティング システムとの間で通信をする場合は、この種類を使用します。外部仮想ネットワークは、物理ネットワーク アダプターにバインドされ、ホスト オペレーティング システム内に仮想ネットワーク アダプターをもつことができます。外部仮想ネットワークは、バーチャル マシンが境界ネットワーク (スクリーン サブネットまたは DMZ とも呼ばれる) にアクセスして、ホスト オペレーティング システムを公開しないようにするために使用できます。
Virtual Server ホスト上で VMM を使って、Virtual Server での構成と同じように仮想ネットワークを構成できます。VMM では、Virtual Server ホスト上で 2 種類の異なる仮想ネットワークを作成できます。
内部仮想ネットワーク。ネットワーク アダプターが選択されていない場合は、仮想ネットワークに接続されているいずれかのバーチャル マシンが内部バーチャル マシン ネットワークの一部になります。各内部バーチャル マシン ネットワークは、他のすべての内部バーチャル マシン ネットワークと完全に分離されています。
外部仮想ネットワーク。物理コンピューターでネットワーク アダプターが選択されていない場合は、仮想ネットワークに接続されているいずれかのバーチャル マシンから、物理アダプターが接続されているネットワークにアクセスすることができます。
Virtual Server での仮想ネットワークの構成についての詳細は、「Creating a virtual network (仮想ネットワークの作成)」(http://go.microsoft.com/fwlink/?LinkId=158803) を参照してください。
VMware では、仮想ネットワークのことを仮想スイッチとも呼びます。ESXServer ホスト上の異なる種類の仮想ネットワークは、VMM では次のように表示されます。
プライベート仮想ネットワーク。VMM では次の 2 種類の構成がプライベート仮想ネットワークとして表示されます。
ホスト上の各バーチャル マシンが互いの間でのみ通信を行うように、バーチャル マシン ポート グループにのみ接続されているすべての仮想ネットワーク アダプターで構成された仮想スイッチ。これは、VMM のプライベート仮想ネットワークと同様です。
ホスト上の各バーチャル マシンが互いの間で、またホストとの間で通信を行えるように、バーチャル マシンのポート グループに接続されている仮想ネットワーク アダプターで構成された仮想スイッチ。これは、VMM の内部仮想ネットワークと同様です。
外部仮想ネットワーク。ホスト上のバーチャル マシンの相互間の通信、および、外部に配置されているサーバーとの通信を可能にするため、バーチャル マシンのポート グループに接続されたすべての仮想ネットワーク アダプターで構成された仮想スイッチ、および物理ネットワーク アダプターにバインドされた仮想スイッチは、VMM では外部仮想ネットワークとして表示されます。オプションで、仮想スイッチをサービス コンソールのポート グループに接続することもできます。これは、VMM の外部仮想ネットワークと同様です。
最高レベルの制御を行うために、VMware では新しい仮想ネットワークを作成して構成します。ESX Server ホスト上の仮想ネットワークの詳細については、VMware のドキュメントを参照してください。
